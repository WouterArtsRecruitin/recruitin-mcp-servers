## POST TEMPLATES PER TYPE

### TYPE 1: CONTRARIAN TAKE (Hoogste engagement)

**Structuur:**
```
[BOLD STATEMENT - controversieel maar waar]

[Lege regel]

[3-5 regels uitleg met specifieke data/voorbeeld]

[Lege regel]

[Plot twist of nuance]

[Lege regel]

[Call-to-engagement: vraag die uitnodigt tot reactie]
```

**Voorbeeld:**
```
Recruiters die "culture fit" als selectiecriterium gebruiken zijn lui.

Ik zei wat ik zei.

In de afgelopen 6 maanden zag ik 47 kandidaten afgewezen worden
op "culture fit". Geen van de hiring managers kon uitleggen
wat dat precies betekende.

Het echte probleem? Ze hadden geen duidelijke job requirements.
"Culture fit" is de vuilnisbak voor ongedefinieerde criteria.

Weet je wat wél werkt? "Culture add" - wat brengt iemand dat
jullie team nog NIET heeft?

Wanneer heb jij voor het laatst iemand aangenomen die helemaal
NIET in het plaatje paste?
```

---

### TYPE 2: DATA STORY (Authority builder)

**Structuur:**
```
[Specifiek getal + context]

[Lege regel]

[Wat betekent dit getal - 2-3 zinnen]

[Lege regel]

[3-5 bullet points met insights]

[Lege regel]

[Conclusie + actie voor lezer]
```

**Voorbeeld:**
```
€519.000 aan deals. Vast in Stage 2. Al 3 maanden.

Dit is geen fictief voorbeeld. Dit is mijn Pipedrive dashboard
van deze ochtend.

Wat ik leerde van het analyseren van 65 vastgelopen deals:

→ 73% had geen duidelijke "next step" gedefinieerd
→ 41% wachtte op informatie die nooit expliciet gevraagd was
→ 28% had simpelweg geen follow-up gehad in 14+ dagen

De fix was pijnlijk simpel: elke deal krijgt nu een
"blokkade reden" veld. Verplicht.

Resultaat na 2 weken: 12 deals door naar Stage 3.

Wat is jouw grootste pipeline-killer op dit moment?
```

---

### TYPE 3: BEHIND-THE-SCENES (Authenticiteit)

**Structuur:**
```
[Eerlijke bekentenis of kwetsbaar moment]

[Lege regel]

[Context - wat gebeurde er]

[Lege regel]

[Wat je leerde - concreet]

[Lege regel]

[Hoe je het nu anders doet]
```

**Voorbeeld:**
```
Ik verloor vorige maand een kandidaat aan een concurrent.

Niet omdat zij meer betaalden.
Niet omdat de rol beter was.

Maar omdat ik 3 dagen te laat reageerde op zijn vraag
over de pensioenregeling.

Drie. Dagen.

In die tijd had de concurrent al een video-call geregeld
met hun HR-manager om alles uit te leggen.

Mijn les: snelheid is niet "nice to have" in recruitment.
Het IS het product.

Sindsdien: max 4 uur response time. Geen uitzonderingen.
Zelfs als het antwoord is "ik zoek het uit en kom morgen
bij je terug."

Wat is jullie interne SLA voor kandidaat-communicatie?
```

---

### TYPE 4: HOW-TO / WAARDE POST (Thought leadership)

**Structuur:**
```
[Problem statement die herkenning triggert]

[Lege regel]

[Simpele oplossing in 1 zin]

[Lege regel]

[Stap-voor-stap breakdown - genummerd of bullets]

[Lege regel]

[Bewijs dat het werkt - resultaat]

[Lege regel]

[CTA: save, deel, of vraag]
```

**Voorbeeld:**
```
"We krijgen geen reacties op onze vacatures."

Dit hoor ik elke week. En elke week is de oorzaak hetzelfde.

De eerste 2 zinnen van je vacaturetekst zijn waardeloos.

Hier is de fix:

1. Schrap alles vóór "Wat ga je doen"
2. Start met de meest interessante taak (niet de saaiste)
3. Voeg een specifiek getal toe (team size, budget, impact)
4. Eindig de eerste paragraaf met waarom dit NU urgent is

Voor/na voorbeeld:

❌ "Wij zijn een innovatief bedrijf in de maakindustrie
   met een rijke historie van 45 jaar..."

✅ "Je eerste project: de productielijn voor onze grootste
   klant (€2.3M/jaar) 15% efficiënter maken. Het team van
   6 engineers wacht op jouw lead."

Resultaat bij laatste 5 klanten: +340% meer sollicitaties.

Save deze post. Je gaat hem nodig hebben.
```

## LINKEDIN ALGORITME OPTIMALISATIE

### Post Timing (Nederlandse markt)

| Dag | Beste tijd | Type content |
|-----|------------|--------------|
| Dinsdag | 08:00-09:00 | Data/insights |
| Woensdag | 12:00-13:00 | Controversy/debate |
| Donderdag | 08:00-09:00 | How-to/waarde |
| Vrijdag | 10:00-11:00 | Behind-scenes/light |

### Algoritme Triggers (2025)

**DO:**
- [ ] Eerste 3 regels = hook (mensen moeten "meer weergeven" klikken)
- [ ] Reageer op ELKE comment binnen 1 uur
- [ ] Gebruik 3-5 relevante hashtags (onderaan, niet in tekst)
- [ ] Tag max 3 personen (alleen als relevant)
- [ ] Post native content (geen externe links in post zelf)
- [ ] Vraag een echte vraag (geen retorische)
- [ ] Lengte: 1200-1500 karakters (sweet spot)

**DON'T:**
- [ ] Links in de post (→ zet in eerste comment)
- [ ] Meer dan 5 hashtags
- [ ] Engagement pods (LinkedIn detecteert dit)
- [ ] "Eens?" of "Deel dit" als CTA
- [ ] Stock photos (algoritme downrankt)
- [ ] Posts bewerken binnen 10 minuten na publicatie

### Engagement Boosters

```
HOOK FORMULES:

"Unpopular opinion: [standpunt]"
"Ik maakte een fout van €[bedrag]. Dit leerde ik."
"[Cijfer]% van [groep] doet dit fout."
"Stop met [veelvoorkomende actie]. Doe dit in plaats."
"De waarheid over [topic] die niemand je vertelt."
"Ik analyseerde [X] [onderwerp]. Dit viel op."
```

## ARTIKEL/BLOG STRUCTUUR

### Voor recruitin.nl + LinkedIn Articles

**Lengte:** 1500-2500 woorden
**Structuur:**

```markdown
# [Titel: Specifiek + Benefit of Getal]
Voorbeeld: "Hoe Wij 47 Tech Vacatures Invulden in 90 Dagen (En Wat We Fout Deden)"

## Lead Paragraph (Hook)
[2-3 zinnen die het probleem benoemen + hint naar oplossing]
[Waarom zou lezer verder moeten lezen?]

## Het Probleem [of: Waarom Dit Ertoe Doet]
[Context, data, herkenbare frustraties]
[Concrete voorbeelden uit eigen praktijk]

## De Oplossing [of: Wat Wij Anders Deden]
[Stap-voor-stap of framework]
[Screenshots/voorbeelden waar relevant]

## Resultaten [met specifieke cijfers]
[Voor/na vergelijking]
[Timeline]

## Wat Jij Kunt Doen [Actionable takeaways]
[3-5 concrete acties die lezer morgen kan implementeren]

## Conclusie + CTA
[Samenvatting in 2 zinnen]
[Duidelijke next step: contact, download, comment]
```

### SEO Checklist voor Artikelen

- [ ] Target keyword in titel, H2, eerste 100 woorden
- [ ] Meta description 150-160 karakters met keyword
- [ ] Minimaal 3 interne links naar andere recruitin.nl pagina's
- [ ] 1-2 externe links naar autoritaire bronnen
- [ ] Alt-text op alle afbeeldingen met keyword variatie
- [ ] URL structure: recruitin.nl/blog/[keyword-slug]

## WHITEPAPER/LEAD MAGNET FRAMEWORK

### Structuur (3000-5000 woorden)

```
TITEL: [Specifieke uitkomst] + [Timeframe/Methode]
Voorbeeld: "De Recruitin Methode: Technische Vacatures 2x Sneller Invullen"

COVER PAGE:
- Titel
- Subtitel met benefit
- Recruitin logo
- "Download datum: [maand jaar]"

INHOUDSOPGAVE

1. EXECUTIVE SUMMARY (1 pagina)
   - Probleem in 2 zinnen
   - Oplossing in 2 zinnen
   - Key resultaten/cijfers

2. HET PROBLEEM (2-3 pagina's)
   - Industrie context met data
   - Herkenbare pijnpunten
   - Kosten van niet-handelen

3. DE OPLOSSING (5-7 pagina's)
   - Framework/methode introductie
   - Stap-voor-stap breakdown
   - Per stap: wat, waarom, hoe
   - Voorbeelden en templates

4. CASE STUDIES (2-3 pagina's)
   - 2-3 klantvoorbeelden
   - Situatie → Aanpak → Resultaat format
   - Specifieke cijfers

5. IMPLEMENTATIE (2 pagina's)
   - Quick wins (week 1)
   - Medium-term (maand 1)
   - Volledige implementatie (kwartaal 1)

6. OVER RECRUITIN + CTA (1 pagina)
   - Wie zijn wij (3 zinnen)
   - Wat we doen (3 bullets)
   - Volgende stap (gratis consult/audit)
   - Contact info
```
